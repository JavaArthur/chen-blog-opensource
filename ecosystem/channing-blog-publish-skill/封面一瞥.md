# 第一步：深度内容分析与吸引力策略

## 🎯 核心目标
**生成的封面必须让用户在信息流中停下来并点击！**

## 分析流程

### 1. 阅读并理解文章内容
- 仔细阅读用户提供的文章
- 提取核心主题、关键信息、情绪基调
- 识别目标受众和他们的需求

### 2. **品牌与真实形象识别**

**核心原则：真实的品牌形象比抽象图标更有辨识度和可信度**

识别文章中提到的：
- **产品/工具名称**：OpenClaw、Claude、ChatGPT、Notion、Figma 等
- **品牌吉祥物**：OpenClaw 的红色小龙虾、GitHub 的章鱼猫等
- **标志性视觉元素**：特定的 Logo、配色方案、UI 界面
- **公司/平台名称**：Google、Apple、微软等

**提取策略：**

A. **如果文章重点讨论某个具体工具/产品**（如 OpenClaw、Claude、Notion）：
   - 记录产品名称和已知的视觉特征
   - 例如："OpenClaw - 红色小龙虾吉祥物"
   - 例如："Claude - 橙色 Logo，AI 助手"
   - 例如："Notion - 黑白极简 Logo"

B. **如果文章对比多个工具**（如 A vs B）：
   - 记录所有相关产品及其视觉特征
   - 准备在封面中展示对比/并列关系

C. **如果文章提到数量/列表**（如"五个 Agent"、"三种方法"）：
   - 记录具体数量和每个项目的名称
   - 准备用真实形象替代抽象图标

**输出格式：**
```
🏷️ **识别到的品牌/产品：**
- [产品名称]：[已知视觉特征/吉祥物描述]
- [产品名称]：[已知视觉特征/吉祥物描述]
```

### 3. 吸引力分析（最关键！）

回答以下问题：

**A. 用户为什么要点击这篇文章？**
- 能解决什么具体问题？
- 能获得什么实际价值？
- 能满足什么好奇心？
- 能引发什么情绪共鸣？

**B. 文章最吸引人的点是什么？**
- 有没有反常识的发现？
- 有没有独特的方法或工具？
- 有没有戏剧性的故事或转折？
- 有没有能引发共鸣的痛点？

**C. 目标受众的心理状态？**
- 他们在寻找什么？
- 他们在焦虑什么？
- 他们会对什么感兴趣？

### 4. 选择 Hook 策略

基于上述分析，选择最合适的吸引力策略：

- **curiosity-gap（好奇缺口）** - 适合：揭秘、发现、实验、反常识内容
  - 视觉：问号、未完成图形、悬念元素
  - 情绪：神秘、好奇、期待

- **pain-point（痛点共鸣）** - 适合：解决方案、教程、问题分析
  - 视觉：困境场景、对比前后、挣扎元素
  - 情绪：共鸣、理解、希望

- **benefit-driven（利益驱动）** - 适合：工具推荐、方法论、技巧分享
  - 视觉：数字、箭头、成果展示、清晰层次
  - 情绪：专业、可信、实用

- **contrast-shock（反差冲击）** - 适合：观点文章、对比分析、批判性内容
  - 视觉：强烈对比色、分屏、意外元素
  - 情绪：冲击、思考、惊讶

- **emotional-trigger（情绪触发）** - 适合：故事、观点、争议话题
  - 视觉：人物表情、戏剧性光影、情绪化色彩
  - 情绪：强烈、共鸣、激动

- **social-proof（社会认同）** - 适合：热门话题、趋势分析
  - 视觉：人群、热度符号、流行元素
  - 情绪：从众、认同、参与

- **scarcity-urgency（稀缺紧迫）** - 适合：限时内容、独家信息
  - 视觉：倒计时、限量标识、紧迫符号
  - 情绪：紧迫、稀缺、行动

### 5. 配套维度选择

基于 Hook 策略，自动选择其他维度：

**Type（构图类型）：**
- curiosity-gap → scene 或 metaphor（叙事感、神秘感）
- pain-point → hero 或 conceptual（冲击力、清晰展示）
- benefit-driven → conceptual（信息图式、清晰层次）
- contrast-shock → typography 或 hero（文字冲击、视觉对比）
- emotional-trigger → metaphor 或 scene（情感隐喻、氛围场景）

**Palette（色板）：**
核心原则：**在信息流中形成视觉反差，高饱和度优先**
- curiosity-gap / contrast-shock → duotone 或 vivid（高对比）
- pain-point / emotional-trigger → dark 或 warm（情绪化）
- benefit-driven → cool + vivid 或 elegant（专业但醒目）
- social-proof → vivid 或 retro（热闹感）
- scarcity-urgency → vivid（高饱和度、紧迫感）

**特别注意：如果识别到品牌，优先使用品牌色**
- OpenClaw → 红色主色调
- Claude → 橙色/棕色主色调
- Notion → 黑白极简
- GitHub → 黑色/紫色

**Rendering（渲染风格）：**
- benefit-driven → digital 或 flat-vector（清晰直接）
- curiosity-gap → painterly 或 hand-drawn（神秘质感）
- contrast-shock → screen-print 或 digital（强烈对比）
- emotional-trigger → painterly 或 hand-drawn（情感质感）

**Mood（情绪强度）：**
- curiosity-gap / contrast-shock / scarcity-urgency → **bold**（强烈）
- pain-point / emotional-trigger → balanced 或 bold
- benefit-driven → bold（信息流中需要高对比）

### 6. 确认宽高比

询问用户："需要什么宽高比？"
- **16:9** - 公众号、博客、横版社交媒体（推荐）
- **1:1** - Instagram、小红书方图
- **3:4** - 竖版海报、手机屏幕
- **2.35:1** - 电影感宽屏

## 输出格式

向用户展示分析结果：

```
📊 **内容分析完成！**

🏷️ **识别到的品牌/产品：**
- [列出识别到的品牌及其视觉特征]

🎯 **吸引力策略**
Hook: [策略名称]
核心吸引点：[一句话说明为什么用户会点击]

🎨 **视觉配置**
- Type: [type]
- Palette: [palette] + [品牌色]
- Rendering: [rendering]
- Mood: [mood]

💡 **设计思路**
[简要说明视觉如何配合吸引力策略，以及如何使用真实品牌形象]

📐 **请选择宽高比：**
- 16:9（公众号/博客推荐）
- 1:1（小红书/Instagram）
- 3:4（竖版海报）
- 其他比例
```

等待用户确认宽高比后，进入第二步。
# 第二步：生成高吸引力封面（专业设计师视角）

## 提示词生成原则

### 🔥 核心要求（按优先级）

1. **吸引力第一** - 必须在 3 秒内抓住注意力
2. **真实形象优先** - 使用真实的品牌吉祥物/Logo，而非抽象图标
3. **信息流优化** - 在小屏幕缩略图中依然醒目
4. **情绪共鸣** - 触发用户的心理触点
5. **价值清晰** - 让用户知道"点进去能得到什么"

### 🎨 文字使用决策框架（顶级设计师视角）

**核心原则：文字是否使用取决于内容本身的特点，而非技术限制**

#### 决策流程（按优先级判断）：

**✅ 必须使用文字的场景（优先级高→低）：**

1. **标题本身就是核心吸引力**
   - 标题包含创意比喻、双关语、反常识表达
   - 例如："赛博阿兹海默症"、"AI 的中年危机"、"代码界的米其林"
   - 判断标准：去掉标题后，吸引力下降 >50%
   - **设计策略**：标题作为主视觉焦点（40-60%），品牌形象为辅助

2. **数字/对比是核心卖点**
   - 标题强调具体数字、对比关系
   - 例如："5 个 Agent"、"提升 10 倍效率"、"A vs B"
   - 判断标准：数字/对比关系是用户点击的主要动机
   - **设计策略**：大号数字 + 品牌形象 + 简短说明文字

3. **技术/教程类内容需要明确主题**
   - 目标受众是开发者、专业人士
   - 需要快速传达"这篇文章讲什么"
   - 例如："Graphiti 完整实战"、"Neo4j 配置指南"
   - 判断标准：受众习惯直接、清晰的信息表达
   - **设计策略**：清晰标题 + 技术视觉元素（代码、图表、品牌 Logo）

4. **观点/争议类内容需要文字冲击**
   - 标题本身具有争议性、挑战性
   - 例如："AI 不会取代程序员"、"别再用 XX 了"
   - 判断标准：文字的冲击力 > 视觉隐喻的冲击力
   - **设计策略**：大胆的文字排版 + 对比色 + 情绪化视觉元素

**🎯 可以不用文字的场景：**

1. **品牌形象本身就是故事**
   - 文章围绕某个知名产品/工具展开
   - 品牌吉祥物/Logo 能直接传达主题
   - 例如：OpenClaw 小龙虾的形象本身就能讲故事
   - **设计策略**：品牌形象作为绝对主角（70%+）

2. **视觉隐喻足够强烈**
   - 能用一个清晰的视觉比喻讲完整个故事
   - 例如：碎片→完整、黑暗→光明、混乱→秩序
   - **设计策略**：强烈的视觉对比 + 品牌色

3. **情绪/氛围类内容**
   - 文章重点在于传达某种感受
   - 视觉氛围比文字描述更有力
   - **设计策略**：情绪化色彩 + 场景化构图

#### 文字设计最佳实践（当决定使用文字时）：

**📝 中文文字处理：**
- **字数控制**：标题 6-12 字为最佳（一行显示完整）
- **字体选择**：粗体、高对比度、易识别
- **位置布局**：
  - 上方居中（经典布局，适合大部分场景）
  - 左侧垂直（现代感，适合年轻受众）
  - 分散排列（创意感，适合设计/艺术类）
- **色彩对比**：文字与背景对比度 ≥7:1（WCAG AAA 标准）
- **层次设计**：主标题 + 副标题（可选）+ 视觉元素

**🔤 英文/数字处理：**
- 更适合作为装饰性元素或辅助信息
- 可以更大胆、更艺术化
- 适合与中文混排（如品牌名 + 中文说明）

**⚡ 文字与视觉的黄金比例：**
- **文字主导型**（标题是核心吸引力）：文字 50-60% + 视觉 40-50%
- **平衡型**（文字和视觉同等重要）：文字 40% + 视觉 40% + 留白 20%
- **视觉主导型**（品牌形象是核心）：视觉 60-70% + 文字 30-40%

### 品牌形象库（常见产品的视觉特征）

**AI 工具类：**
- **OpenClaw**：红色小龙虾吉祥物，可爱卡通风格，有大钳子和触角
- **Claude**：橙棕色调，简洁 AI 助手形象
- **ChatGPT**：绿色/青色 Logo，对话气泡
- **Midjourney**：帆船 Logo，蓝紫色调
- **Stable Diffusion**：抽象渐变 Logo

**生产力工具类：**
- **Notion**：黑白极简 Logo，方块图标
- **Obsidian**：紫色宝石/水晶图标
- **Roam Research**：蓝色网状图标
- **Figma**：彩色几何图形 Logo

**开发工具类：**
- **GitHub**：黑色章鱼猫（Octocat）吉祥物
- **VS Code**：蓝色 Logo
- **Docker**：蓝色鲸鱼吉祥物
- **Neo4j**：蓝绿色节点网络图谱

**其他：**
- **Apple**：银色/白色苹果 Logo，极简风格
- **Google**：四色 Logo（蓝红黄绿）

### 质量检查清单（更新版）

生成提示词前，确保：

✅ **文字决策检查（新增！最重要）：**
- [ ] 已基于决策框架判断是否需要文字
- [ ] 决策理由清晰且符合专业设计师视角
- [ ] 如果使用文字，已确定文字-视觉比例
- [ ] 如果使用文字，字数控制在 6-12 字
- [ ] 如果使用文字，已设计清晰的排版布局
- [ ] 如果不用文字，已确保纯视觉能传达核心信息

✅ **品牌真实性检查：**
- [ ] 如果文章提到具体品牌，必须使用真实的吉祥物/Logo 描述
- [ ] 品牌形象描述具体且可识别（颜色、形状、特征）
- [ ] 不使用"抽象图标"、"通用符号"等模糊描述
- [ ] 品牌色融入整体配色方案

✅ **吸引力检查：**
- [ ] 主视觉焦点明确且占比 40-60%
- [ ] 色彩饱和度高，能在信息流中脱颖而出
- [ ] 有明确的"钩子"元素（文字/视觉隐喻/品牌形象/色彩对比/数字）
- [ ] 构图有张力和动态感

✅ **Hook 策略检查：**
- [ ] 视觉元素与 Hook 策略完全匹配
- [ ] 能在 3 秒内传达核心吸引点
- [ ] 移动端缩略图依然清晰可见

✅ **避免错误：**
- [ ] 不要过于柔和的色彩（会被淹没）
- [ ] 不要过于抽象（用户看不懂）
- [ ] 不要过于复杂（3 秒内无法理解）
- [ ] 不要过于"艺术化"（失去吸引功能）
- [ ] **不要使用通用图标代替真实品牌形象**
- [ ] **不要因为技术顾虑而逃避使用文字**
