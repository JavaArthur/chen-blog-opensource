# 第一步：深度文章分析与段落拆解

## 🎯 核心目标

**为文章的每个关键段落生成风格匹配、见图知意的配图，让读者通过图片就能理解核心内容。**

## 分析流程

### 1. 阅读并理解文章全貌

- 仔细阅读用户提供的文章

- 提取文章主题、核心观点、整体基调

- 识别目标受众和内容类型（教程/观点/故事/科普等）

### 2. 文章结构拆解

识别文章的主要段落结构：

- **标题/引言**：是否需要封面图？

- **核心段落**：每个二级标题下的主要内容

- **总结/结尾**：是否需要总结图？

**输出格式：**

```

📝 **文章结构：**

- 段落1：[标题] - [内容类型]

- 段落2：[标题] - [内容类型]

- 段落3：[标题] - [内容类型]

...

```

### 3. 内容类型识别（最关键！）

为每个段落识别其内容类型，这决定了图片的布局和表达方式：

**A. 流程型（flow/steps）**

- 特征：包含步骤、流程、时间线、操作指南

- 例如："Week 1: 观察 → Week 2: 提炼 → Week 3: 自动化"

- 视觉策略：箭头流程图、时间轴、步骤卡片

- 布局：flow（流程图）或 list（步骤列表）

**B. 对比型（comparison）**

- 特征：包含对比、优劣、前后差异、A vs B

- 例如："错误用法 vs 正确用法"、"好习惯 vs 坏习惯"

- 视觉策略：左右分屏、对比色、对立元素

- 布局：comparison（对比图）

**C. 列表型（list）**

- 特征：包含要点列举、TOP N、清单、推荐

- 例如："三个问题"、"五个方法"、"必备工具"

- 视觉策略：编号卡片、图标列表、清单式排版

- 布局：list（列表图）或 dense（密集信息图）

**D. 概念型（conceptual）**

- 特征：解释概念、理论、原理、金句

- 例如："注意力残余"、"提问能力金字塔"

- 视觉策略：隐喻图形、概念可视化、金句排版

- 布局：sparse（极简金句）或 balanced（概念图）

**E. 故事/场景型（scene）**

- 特征：讲述案例、故事、场景描述

- 例如："我同事小王的例子"

- 视觉策略：场景插画、人物故事、情境再现

- 布局：balanced（场景图）

### 4. 选择配图数量

基于文章长度和结构，决定生成几张图：

- **3张图**：适合短文章（3-4个核心段落）

  - 封面图 + 2个核心段落配图

  - 或 3个核心段落配图

- **4张图**：适合中等长度文章（4-5个核心段落）

  - 封面图 + 3个核心段落配图

  - 或 4个核心段落配图

- **5张图**：适合长文章（5+个核心段落）

  - 封面图 + 4个核心段落配图

  - 或 5个核心段落配图（包含总结图）

**优先级原则：**

- 优先为包含具体方法、步骤、对比的段落配图

- 纯理论阐述、过渡段落可以跳过

- 确保每张图都有明确的信息价值

### 5. 整体风格选择

基于文章主题和目标受众，选择统一的视觉风格：

**风格维度（Style）：**

- **cute（可爱）**：适合生活方式、日常分享、轻松话题

  - 色彩：粉色、浅紫、柔和色调

  - 元素：圆润形状、卡通图标、可爱插画

  

- **fresh（清新）**：适合健康、环保、简洁生活、正能量

  - 色彩：绿色、蓝色、白色为主

  - 元素：植物、自然、清爽图标

  

- **warm（温暖）**：适合家庭、情感、舒适、治愈话题

  - 色彩：暖黄、橙色、棕色

  - 元素：温馨场景、柔和光影

  

- **minimal（极简）**：适合专业、商务、高端、理性内容

  - 色彩：黑白灰、单一强调色

  - 元素：几何图形、大量留白

  

- **retro（复古）**：适合怀旧、文化、故事、特定年代话题

  - 色彩：复古色调、做旧效果

  - 元素：老照片质感、复古装饰

- **alert（警示）**：适合避坑指南、注意事项、重要提醒

  - 色彩：红色、黄色、高对比

  - 元素：警示符号、醒目标识

**布局密度（Layout Density）：**

- **sparse（稀疏）**：极简风格，大量留白，适合金句、单一概念

- **balanced（平衡）**：信息适中，适合大部分场景

- **dense（密集）**：信息量大，适合知识总结、干货合集

### 6. 输出分析结果

向用户展示完整的配图方案：

```

[一句话概括文章主题]

🖼️ **配图方案（共 X 张）**

**图1：[段落标题]**

- 内容类型：[类型]

- 核心信息：[这张图要传达什么]

- 布局方式：[layout]

- 视觉策略：[具体怎么表现]

**图2：[段落标题]**

- 内容类型：[类型]

- 核心信息：[这张图要传达什么]

- 布局方式：[layout]

- 视觉策略：[具体怎么表现]

  - 想要不同的风格？

  - 想要更多/更少的图片？

  - 想要调整某张图的内容？

```

等待用户确认后，进入第二步。

# 第二步：生成风格统一的配图（信息图设计师视角）

## 提示词生成原则

### 🔥 核心要求（按优先级）

1. **信息传达第一**：图片必须清晰传达段落的核心信息

2. **见图知意**：用户看图就能理解内容，无需阅读文字

3. **风格统一**：所有图片保持一致的视觉风格

4. **信息图化**：采用信息图（infographic）的设计思路，而非纯装饰插画

5. **中文友好**：必须包含清晰的中文文字说明

### 🎨 提示词结构（信息图专用）

为每张图生成提示词时，使用以下结构：

```

🎯 CONTENT STRATEGY:

Paragraph Title: [段落标题]

Core Message: [这张图要传达的核心信息，一句话]

Content Type: [flow/comparison/list/conceptual/scene]

Key Information: [必须包含的关键信息点，3-5个]

🎨 VISUAL STYLE (统一风格):

Style: [cute/fresh/warm/minimal/retro/alert]

Layout Density: [sparse/balanced/dense]

Color Palette: [具体色彩方案，与整体风格一致]

Illustration Style: [插画风格描述]

📐 LAYOUT & COMPOSITION:

Layout Type: [基于内容类型选择]

- flow: 箭头流程图，从左到右或从上到下

- comparison: 左右分屏对比，中间分割线

- list: 编号列表，垂直或网格排列

- sparse: 极简布局，大量留白，单一焦点

- balanced: 平衡布局，图文结合

- dense: 密集信息图，多模块组合

Composition Details: [具体的布局描述]

- 主标题位置和样式

- 内容模块的排列方式

- 图标和插画的位置

- 留白和分割的处理

✍️ TEXT ELEMENTS (关键！):

Text Decision: MUST USE（信息图必须包含文字）

Text Content:

- 主标题：[段落标题，6-12字]

- 内容文字：[关键信息点的文字说明，每条5-15字]

- 装饰文字：[可选，如标签、注释]

Text Layout:

- 主标题：[位置] - [字体大小] - [颜色]

- 内容文字：[排列方式] - [字体大小] - [颜色]

- 文字层次：清晰的主次关系

Typography Style: [字体风格，如手写体/黑体/宋体/圆体]

🎨 VISUAL ELEMENTS:

Icons & Illustrations: [需要的图标和插画元素]

- [元素1]：[描述和用途]

- [元素2]：[描述和用途]

- ...

Decorative Elements: [装饰性元素]

- 边框、分割线、背景纹理等

Color Usage: [色彩的具体运用]

- 主色：[用于哪里]

- 辅色：[用于哪里]

- 强调色：[用于哪里]

⚡ INFOGRAPHIC REQUIREMENTS:

- Clear information hierarchy（清晰的信息层级）

- Visual metaphors for abstract concepts（抽象概念的视觉隐喻）

- Consistent icon style throughout（统一的图标风格）

- Readable Chinese text with proper font（清晰可读的中文字体）

- Balanced composition with breathing room（平衡的构图和留白）

- Professional but approachable design（专业但亲切的设计）

🔧 TECHNICAL:

- Aspect ratio: 1:1（方图，适合文章配图）

- Style: [整体风格]

- Layout: [布局类型]

- Text rendering: Chinese, [字数], [字体风格]

- High contrast for readability（高对比度，易读性优先）

```

### 📋 内容类型对应的视觉策略

**Flow（流程型）：**

```

视觉元素：

- 编号圆圈或方框（1、2、3...）

- 箭头连接各步骤

- 每步骤配图标和文字说明

- 时间线或进度条（可选）

布局：

- 横向流程：左→右，适合3-4步

- 纵向流程：上→下，适合4-6步

- 循环流程：圆形排列，适合循环过程

参考示例：

"Week 1: 观察" → "Week 2: 提炼" → "Week 3: 自动化" → "Week 4: 优化"

每个步骤用圆形图标 + 标题 + 简短说明

```

**Comparison（对比型）：**

```

视觉元素：

- 左右分屏，中间分割线或 VS 标识

- 对比色（如红色 vs 绿色，暖色 vs 冷色）

- 对立的图标或表情（✓ vs ✗，😊 vs 😞）

- 每侧列出对比项

布局：

- 标题居中或居上

- 左右各占50%

- 每侧3-5个对比项

- 对应位置对齐

参考示例：

"好习惯 vs 坏习惯"

左侧绿色：早睡早起、均衡饮食、规律运动...

右侧红色：熬夜晚睡、暴饮暴食、久坐不动...

```

**List（列表型）：**

```

视觉元素：

- 编号（1、2、3... 或 ①②③）

- 统一的图标或插画

- 每项的标题和说明文字

- 装饰性边框或卡片

布局：

- 垂直列表：适合3-5项

- 网格布局：适合4-6项（2x2 或 2x3）

- 每项包含：编号 + 图标 + 标题 + 说明

参考示例：

"TOP 5 好物推荐"

1. 高间隙咖啡机 - 快速出杯，一键操作

2. 舒缓眼罩套装 - 缓解疲劳，温热舒适

...

```

**Conceptual（概念型）：**

```

视觉元素：

- 核心概念的视觉隐喻

- 简洁的图形和图标

- 关键词或金句

- 概念之间的关系线

布局：

- sparse: 大标题 + 核心图形 + 简短说明

- balanced: 中心概念 + 四周扩展要素

- 金字塔、圆环、树状图等结构

参考示例：

"提问能力金字塔"

三层金字塔结构：

顶层：Level 3 高手 - [描述]

中层：Level 2 进阶 - [描述]

底层：Level 1 新手 - [描述]

```

**Scene（场景型）：**

```

视觉元素：

- 场景化的插画

- 人物或拟人化角色

- 环境和道具

- 对话框或思考泡泡（可选）

布局：

- 场景作为背景

- 人物/角色作为主体

- 文字说明融入场景

- 叙事性的构图

参考示例：

"小王的故事"

办公场景，人物坐在电脑前，周围飘着各种任务图标

对话框："每周花6小时整理周报..."

```

## 生成图片

按照第一步确定的方案，依次生成每张图片：

1. 为每张图生成完整的提示词（按照上述结构）

2. 使用 imageGenerate 工具生成图片：

   - model: gemini-3.1-flash-image-preview

   - aspect_ratio: 1:1

   - prompt: [完整的提示词]

   - title: [段落标题]

3. 生成时注意：

   - 所有图片使用相同的 Style 参数

   - 色彩方案保持一致

   - 插画风格保持统一

   - 字体风格保持一致
